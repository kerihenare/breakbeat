## Implementation summary

The Breakbeat app was built task-by-task following `docs/2-plan.md` (Tasks 1–17), with the design spec in `docs/1-spec.md` as the source of truth.

### Key implementation decisions

**Structured outputs for LLM calls (resolve + classify)**
Both the Resolve cascade's LLM fallback and the Classify pass use Anthropic structured outputs (`output_format: json_schema`). The Resolve fallback is constrained to `{ candidate_index: 0..4 | "none" }` — it can only pick from the closed list of domains we hand it, so it cannot hallucinate a domain. The Classify pass uses a per-result schema `{ id, content_type, exclude, confidence }` with no free-text reasoning field. This collapses the prompt-injection blast radius: a hostile snippet in a Tavily result can at worst cause its own misclassification, which soft exclusion makes visible and recoverable.

**IPv6 ULA gap fixed in SSRF guard**
The SSRF guard in `src/jobs/resolve.ts` rejects private IPv4 ranges (10.x, 172.16–31.x, 192.168.x) and loopback/link-local, and also explicitly rejects IPv6 Unique Local Addresses (`fc00::/7` — the `fd` prefix used by most home/office networks). The initial implementation missed the IPv6 ULA range; it was added to close the gap.

**Body stream cancel on 1MB cap**
The homepage fetch enforces a 1MB response cap. Rather than buffering the full response and discarding it, the implementation calls `response.body.cancel()` after reading 1MB to free the connection immediately. This prevents the FIFO-1 queue from being stalled by a slow large response.

**normalizeUrl deduplication from canonical normalize.ts (not duplicated)**
The URL deduplication key (`normalizeUrl`) and title Collapse key (`normalizeTitle`) both live exclusively in `src/filter/normalize.ts`. The Resolve stage, Search stage, and Filter stage all import from this single module. No normalization logic is duplicated across files.

**Outlet-suffix stripping uses full-title ≥25 check (pragmatic resolution of spec ambiguity)**
The spec says to strip a trailing ` | X` / ` – X` / ` - X` segment "only if the remaining title is still ≥ 25 chars." The spec's own example — `Acme - The Real Story` — has a remaining title of 16 chars after stripping "The Real Story" (if read as suffix). The implementation applies the ≥25 check to the title *after* stripping the suffix segment; `Acme - The Real Story` survives intact because "Acme" (4 chars) < 25. The 25-char guard prevents collapsing genuinely distinct short titles and resolves the spec's own example correctly.

**Nunjucks custom `where`/`whereNot` filters (native selectattr broken for equality)**
Nunjucks's built-in `selectattr` filter uses `===` strict equality comparison, which fails for string-to-string comparison in the version used here. The `src/main.ts` bootstrap registers custom `where(arr, key, val)` and `whereNot(arr, key, val)` environment filters using `==` loose equality. These are used in `src/views/_job.njk` to group results by `content_type` and exclusion rows by `exclusion_code`.
